## 2. 상세 요구사항

### 2.1. 회원가입 `REQ-AUTH-001`

- **업무구분(대)**: 회원관리
- **업무구분(소)**: 등록
- **우선순위**: P0
- **비고**: https://datatracker.ietf.org/doc/html/rfc5321#section-4.5.3.1.3

#### 요구사항 내용

**[기본 흐름]**
• 사용자는 이메일·비밀번호·닉네임을 입력하여 회원가입할 수 있다.
• 회원가입 전 개인정보 수집·이용 동의 및 이용약관 동의가 필요하며, 미동의 시 가입 API 요청을 보내지 않고 가입이 불가하다.
• 필수 입력값과 필수 약관 동의가 모두 충족되면 회원가입을 완료할 수 있다.
• 회원가입 완료시 토큰 5개가 지급된다.


**[이메일]**
• 이메일 형식(RFC 5321) 검증을 수행한다.
• 이메일 중복확인을 진행하며 이미 가입된 이메일로는 인증 코드를 발송할 수 없다.
• 중복확인 완료된 이메일로 무작위 영문 대문자 6자리 인증 코드가 발송되며, 3분 내로 입력 해야한다.
• 인증코드 발송 이후 30초 후 가능하며, 재발급 시 기존 코드는 즉시 만료 처리된다.

**[비밀번호]**
• 영어 대소문자·숫자·특수문자 중 최소 2종류 이상의 조합으로 이루어진 8~16자를 입력받으며 공백은 포함하지 않는다.
• 비밀번호 확인을 통해 입력된 비밀번호와 일치여부를 확인한다.
• 비밀번호 확인 불일치 시 가입 불가하다.

**[닉네임]**
• 사용자는 서비스 내에서 사용할 닉네임을 설정할 수 있다.
• 한글·영문·숫자로 2~8자를 입력받으며 공백은 포함하지 않는다.

**[선택정보]**
• 직업과 생년월일은 입력하지 않아도 회원가입이 가능하다.
• 직업은 목록으로 표시되며 수집한 직업 정보는 사용자 통계 분석 및 AI 학습에 활용된다.
• 생년월일은 년/월/일이 목록으로 표시되며 연령대별 분석과 생일 이벤트에 이용된다.
• AI 활용 선택 동의 여부는 `is_aiconsent` boolean 값으로 저장하며, `false`여도 회원가입이 가능하다.

### 2.2. 로그인 `REQ-AUTH-002`

- **업무구분(대)**: 회원관리
- **업무구분(소)**: 인증
- **우선순위**: P0

#### 요구사항 내용

**[기본 흐름]**
• 가입된 이메일·비밀번호로 로그인할 수 있어야 한다.
• 입력값이 공란일 경우 오류 메시지를 반환한다.
• 미가입 이메일 또는 비밀번호 불일치 시 오류 메시지를 반환한다.
• 로그인 실패 시 이메일 유지, 비밀번호 입력란이 초기화된다.
• 최초 로그인시 캐릭터 생성 모달이 나타나며, 캐릭터를 생성하지 않으면 서비스 이용이 불가능하다.

**[자동 로그인]**
• 사용자의 기기 정보를 기반으로 서버에서 검증 가능한 암호화된 토큰(예: JWT, 세션)을 생성한다.
• 접속 시 저장된 토큰을 확인하고, 유효성 검증 성공 시 로그인 과정을 생략하고 로그인 처리된다.
• 사용자가 비밀번호를 변경하거나 로그아웃한 경우, 기존 기기의 자동 로그인 토큰을 무효화한다.
• 2주간 접속하지 않으면 자동 로그인 토큰을 무효화 처리한다.

### 2.3. 소셜 로그인 `REQ-AUTH-003`

- **업무구분(대)**: 회원관리
- **업무구분(소)**: 인증
- **우선순위**: P1

#### 요구사항 내용

**[기본 흐름]**
• 사용자는 소셜 계정을 통해 간편 로그인/회원가입을 할 수 있다.
• 소셜 로그인 시 이메일을 수집하여 계정을 생성한다.
• 최초 소셜 로그인 시 추가 정보(닉네임, 직업, 생년월일) 입력이 가능하다.
• 이미 가입된 이메일과 동일한 소셜 계정으로 로그인 시 기존 계정과 연동 처리한다.
• 최초 로그인시 캐릭터 생성 모달이 나타난다.

**[지원 플랫폼]**
• Kakao

**[제약 사항]**
• 소셜 로그인 사용자는 서비스 내 비밀번호가 별도로 설정되지 않으며, 비밀번호 재설정을 할 수 없다.

- **업무구분(대)**: 회원관리
- **업무구분(소)**: 인증
- **우선순위**: P1

#### 요구사항 내용

• 사용자는 현재 로그인된 계정에서 로그아웃할 수 있다.
• 로그아웃 시 현재 로그인 세션이 종료되고, 랜딩페이지로 이동한다.

### 2.5. 회원정보 수정 `REQ-AUTH-005`

- **업무구분(대)**: 회원관리
- **업무구분(소)**: 수정
- **우선순위**: P2
- **비고**: 세션 만료 시간(N분) TBD

#### 요구사항 내용

**[회원정보 수정]**
• 마이페이지 진입 후 직업, 닉네임 수정이 가능하다.
• 닉네임 제약 조건은 REQ-AUTH-001과 동일.
• 현재 비밀번호 입력 후 일치시 새 비밀번호로 수정이 가능하다.
• 비밀번호 수정 진행시 비밀번호 제약 조건은 REQ-AUTH-001과 동일하다.
• 새 비밀번호 확인을 통해 입력된 새 비밀번호와 일치여부를 확인한다.
• 비밀번호 수정 후 보안을 위해 로그아웃 처리된다.

**[제약 사항]**
• 소셜로그인 회원의 경우 비밀번호 수정은 불가능하다.

### 2.6. 회원 탈퇴 `REQ-AUTH-006`

- **업무구분(대)**: 회원관리
- **업무구분(소)**: 수정
- **우선순위**: P2
- **비고**: 피드 게시물 탈퇴 후 처리 방침 TBD

#### 요구사항 내용

• 사용자는 현재 비밀번호 입력 후 일치시 회원 탈퇴를 진행할 수 있다.
• 소셜 로그인 사용자의 경우 현재 비밀번호 입력 없이 회원 탈퇴를 진행할 수 있다.
• 탈퇴 진행시 재확인을 위해 "회원 탈퇴"를 입력해야 회원 탈퇴를 진행 가능하다.
• 탈퇴 완료 시 저장된 개인정보(원본이미지, 닉네임, 이메일, 비밀번호)는 삭제 된다.
• 개인정보 이외의 이용했던 로그정보는 삭제되지 않는다.

### 2.7. 비밀번호 찾기 `REQ-AUTH-007`

- **업무구분(대)**: 회원관리
- **업무구분(소)**: 수정
- **우선순위**: P2

#### 요구사항 내용

• 비밀번호 분실 시 활성 이메일 로그인 계정의 가입 이메일을 입력하여 인증 코드를 발급받을 수 있다.
• 미가입 또는 비활성 이메일 입력 시 `404 USER_NOT_FOUND` 오류를 반환하며 인증 코드를 저장하거나 발송하지 않는다.
• 소셜 로그인 계정은 `403 SOCIAL_ACCOUNT_NO_PASSWORD` 오류를 반환하며 인증 코드를 저장하거나 발송하지 않는다.
• 인증 코드는 무작위 영문 대문자 6자리로 이루어져 있으며 입력 가능 시간은 3분이다.
• 코드 재발급은 발송 이후 30초 후 가능하며, 재발급 시 기존 코드는 즉시 만료 처리된다.
• 새 비밀번호 제약 조건은 REQ-AUTH-001과 동일하다.
• 재설정 완료 시 사용된 코드 즉시 만료 처리 후 로그인을 진행할 수 있다.
